ntfs支持及libreoffice安装

2012-07-24
linux

今天在电脑上工作了很久，但是只做了两件事：

  1. 增加centos对ntfs支持；
  2. 安装libreoffice并解决它带来的问题。

今天上午登录博客的时候发现上不去了，再上godaddy的网站也上不去了，于是便用代理上了godaddy把域名解析改到了dnspod上。结果到晚上了还没生效，一会能上一会不能上的。

我的godaddy账号密码保存在ntfs格式的盘里，我又不想重启到win7再复制到fat32格式的盘里，所以我上网寻求让linux支持ntfs的方法。

显然，有人比我想的要多而且长远。网上已经有了相当成熟的方法－－ntfs-3g。安装和使用方法很简单，照着人家写的做就好了，所以这事没用多长时间。

等域名解析生效的功夫，我想离线写一下文章，于是想起了大名鼎鼎的libreoffice。没想到悲剧就这样发生了。

我从官网下了最新的3.5.5版，也遵照官网上的安装步骤进行了安装。然而当我企图从应用程序－办公里打开它时，我能看到的只是一闪而过的欢迎界面，然后就什么都没了。我很自然地想到删除重装，但是依然如此。

于是我开始了漫长的搜索之路，但是一直无果。一位网友的话某种程序上给了我一丝希望，他说不能运行可能是因为不是文件的所有者。果然，我去到lo的目录下看了眼权限，整个目录都是属于root的。于是我在终端里以root身份运行soffice，结果居然运行起来了。我想，如果把目录的权限交给一般用户，那说不定就能运行。之后我便在鸟哥的私房菜里阅读了权限部分的内容，但是无论我把权限改为777还是把拥有者给到一般用户，情况都像之前一样。而且我还发现一个问题，就算我不辞辛苦跑到lo的目录里并在终端以root身份运行，ibus输入法居然说找不到输入窗口。就算我解决了lo的启动问题，我还得再解决输入法的问题。

最终，我实在没办法了，于是选择了放弃。我把装好的lo删掉，并在终端里yum install libreoffice，一切就这么轻松地解决了。虽然版本比官网上的低，但是能正常打开，并且能用ibus输入法输入了，现在的这篇文章就是在lo里完成的。

官方软件仓库里的软件果然是稳定啊，但是我不爽的是这次遇到问题没有解决。
